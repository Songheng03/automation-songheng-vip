## Overview

Agent-Cloud is an **open-source platform**, not a marketplace or directory. It does **not** have a submission portal, app store, or listing system. Instead, integrations are contributed via **GitHub Pull Requests** to the main repository.

This means there is no "submit" action in the traditional sense. To have your service/integration included, you must contribute code.

## Integration Methods

Agent-Cloud supports three ways to integrate:

### 1. Data Source Connectors (260+ built-in)
- Integrations for BigQuery, HubSpot, Salesforce, Postgres, Snowflake, etc.
- To add a new connector: Fork the repo, add your connector following existing patterns, submit a PR

### 2. LLM Providers
- Bring your own LLM (OpenAI, open-source models via Ollama, etc.)
- No submission needed — configure at deployment time

### 3. Tool/API Integrations
- AI agents can access 3rd party APIs
- Add custom tool support via code contribution

## How to Contribute (No Submission Portal)

Since there is no web form or API for submission, the only way to get your MCP server or tool listed/integrated with Agent-Cloud is through open-source contribution:

### Step 1: Fork the Repository
```bash
git clone https://github.com/rnadigital/agentcloud.git
cd agentcloud
git checkout -b feat/my-automaton-integration
```

### Step 2: Implement Integration
- Follow the existing connector patterns in the codebase
- Add your MCP server endpoint as a supported tool/data source
- Write documentation for your integration
- Add tests (if applicable)

### Step 3: Submit a Pull Request
- Push your branch to your fork
- Open a PR against `rnadigital/agentcloud:main`
- Include detailed description of your integration

### Step 4: Maintain
- Respond to reviewer feedback
- Keep your integration updated as the platform evolves
