## 1. Overview

The **README Generator** allows users to paste a code snippet and receive a complete `README.md` document generated by DeepSeek AI. The generated README includes:

- Project name (derived from context or entered)
- Description
- Installation instructions
- Usage examples
- Features list
- Dependencies
- Configuration (where applicable)
- API reference (for libraries)
- **Attribution footer** (always appended): `*README generated by [my-automaton](https://automation.songheng.vip)*`

The service is rate-limited to **3 requests per day per IP address**, reusing the existing `FREE_LIMIT` infrastructure in the gateway.

## 2. API Endpoint: `POST /api/generate-readme`

### 2.2 Request Format

**Content-Type:** `application/json`

**Request Body Schema:**
```json
{
  "code": "string (required) вҖ” The source code snippet to generate a README for. Max 8000 chars.",
  "language": "string (optional) вҖ” Programming language for context (e.g., 'python', 'javascript', 'go'). Auto-detected if omitted.",
  "name": "string (optional) вҖ” Project name override. If omitted, AI derives it from code context."
}
```

**Example Request:**
```json
{
  "code": "def hello(name):\n    return f\"Hello, {name}!\"\n\nif __name__ == '__main__':\n    print(hello('World'))",
  "language": "python",
  "name": "hello-world"
}
```

**cURL Example:**
```bash
curl -X POST https://automation.songheng.vip/api/generate-readme \
  -H 'Content-Type: application/json' \
  -d '{"code":"console.log(\"hello world\")","language":"javascript"}'
```

### 2.3 Response Format

**Success (200 OK):**
```json
{
  "readme": "# hello-world\n\nA JavaScript utility ...\n\n## Installation\n\n```bash\nnpm install\n```\n\n...\n\n---\n*README generated by [my-automaton](https://automation.songheng.vip)*",
  "language": "javascript",
  "timestamp": "2026-06-18T12:00:00.000Z",
  "free_remaining": 2
}
```

| Field | Type | Description |
|-------|------|-------------|
| `readme` | string | The complete README.md content including attribution footer |
| `language` | string | Detected or confirmed programming language |
| `timestamp` | string (ISO 8601) | Server timestamp of generation |
| `free_remaining` | integer | Remaining free generations for this IP today (0-2) |

**Response Headers:**
```

X-RateLimit-Remaining: 2
```

### 2.4 Rate Limiting Logic

Reuses the existing `FREE_LIMIT` infrastructure from `gateway.cjs`:

```js
// FREE_LIMIT is a Map<string, { date: string, count: number }>
// Keyed by IP address (from x-forwarded-for or remoteAddress)
// Limit: 3 requests per day per IP

function checkFreeLimit(ip) {
  const today = new Date().toISOString().split('T')[0];
  const entry = FREE_LIMIT.get(ip);
  if (!entry || entry.date !== today) {
    FREE_LIMIT.set(ip, { date: today, count: 0 });
    return { allowed: true, remaining: 3 };
  }
  if (entry.count >= 3) return { allowed: false, remaining: 0 };
  return { allowed: true, remaining: 3 - entry.count };
}

function incrementFree(ip) {
  const today = new Date().toISOString().split('T')[0];
  const entry = FREE_LIMIT.get(ip);
  if (!entry || entry.date !== today) {
    FREE_LIMIT.set(ip, { date: today, count: 1 });
  } else {
    entry.count++;
  }
}
```

**Important:** The README generator shares the same FREE_LIMIT pool as the `/free/` endpoints. This means 3 total free requests per day across all free services (including README generation). This is the current behavior in `gateway.cjs` as seen in the `handleGenerateReadme` function which calls `checkFreeLimit(ip)` вҖ” the same function used by `handleFree`.

If a separate quota is desired, a dedicated `README_LIMIT` Map should be created instead.

### 2.5 DeepSeek Integration

The handler calls `callAI()` with a carefully crafted system prompt and user message:

```js
async function handleGenerateReadme(req, res) {
  const ip = ipFromReq(req);

  // Rate limit check
  const { allowed, remaining } = checkFreeLimit(ip);
  if (!allowed) {
    res.writeHead(429, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      error: 'Free limit reached (3/day). Try again tomorrow or buy credits at /upgrade.html',
      upgrade: true, limit: 3, reset_date: new Date().toISOString().split('T')[0]
    }));
    return;
  }

  let body = '';
  req.on('data', c => body += c);
  req.on('end', async () => {
    try {
      const input = JSON.parse(body);
      const code = (input.code || '').trim();
      const language = (input.language || '').trim();
      const projectName = (input.name || '').trim();

      if (!code) {
        res.writeHead(400, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ error: 'Missing required field: code' }));
        return;
      }

      if (code.length > 8000) {
        res.writeHead(413, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ error: 'Code exceeds maximum length of 8000 characters' }));
        return;
      }

      // Build the README prompt
      const systemPrompt = `You are a technical documentation expert. Generate a comprehensive README.md in GitHub-flavored Markdown for the provided source code. Structure the README as follows:

1. **Project Title** вҖ” Derived from the code or the provided project name.
2. **Description** вҖ” 1-2 sentences explaining what the project does.
3. **Features** вҖ” Bullet list of key features.
4. **Installation** вҖ” Step-by-step instructions (use generic package manager commands).
5. **Usage** вҖ” Code examples showing how to use the code.
6. **API Reference** вҖ” If the code defines functions/classes, document them.
7. **Dependencies** вҖ” List any obvious dependencies.
8. **License** вҖ” State "MIT License" (default).

IMPORTANT: At the very end of the README, append a horizontal rule and the following attribution line (exactly):
${''}
---
*README generated by [my-automaton](https://automation.songheng.vip)*

Use appropriate markdown formatting: code blocks with language tags, tables where helpful, and clear section headers. Do not wrap the entire output in a code block вҖ” output raw markdown.`;

      const userMessage = projectName
        ? `Project name: ${projectName}\nLanguage: ${language || 'unknown'}\n\nCode:\n\`\`\`${language}\n${code}\n\`\`\``
        : `Language: ${language || 'unknown'}\n\nCode:\n\`\`\`${language}\n${code}\n\`\`\``;

      const result = await callAI([
        { role: 'system', content: systemPrompt },
        { role: 'user', content: userMessage }
      ]);

      if (result.error) {
        res.writeHead(502, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ error: 'AI service temporarily unavailable. Please try again later.' }));
        return;
      }

      incrementFree(ip);

      // Ensure attribution footer is present
      let readme = result;
      const attribution = '\n\n---\n*README generated by [my-automaton](https://automation.songheng.vip)*';
      if (!readme.includes('generated by [my-automaton]')) {
        readme += attribution;
      }

      res.writeHead(200, {
        'Content-Type': 'application/json',
        'X-RateLimit-Limit': 3,
        'X-RateLimit-Remaining': Math.max(0, 3 - (FREE_LIMIT.get(ip)?.count || 0)),
        'X-RateLimit-Reset': new Date().toISOString().split('T')[0]
      });
      res.end(JSON.stringify({
        readme,
        language: language || 'unknown',
        timestamp: new Date().toISOString(),
        free_remaining: Math.max(0, 3 - (FREE_LIMIT.get(ip)?.count || 0))
      }));

    } catch (e) {
      res.writeHead(400, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({ error: 'Invalid request: ' + e.message }));
    }
  });
}
```

### 2.6 Implementation Notes

1. **System prompt engineering:** The system prompt instructs the AI to always include the attribution footer. The handler also appends it as a fallback if the AI omits it.
2. **Error resilience:** If DeepSeek is down, returns 502 with a user-friendly message.
3. **Request validation:** Checks for empty code, max length, and malformed JSON.
4. **Rate limit headers:** Uses standard `X-RateLimit-*` headers for API consumer clarity.
5. **Shared vs. separate rate limiter:** Currently shares FREE_LIMIT with other free endpoints. If isolation is desired, a dedicated `const README_LIMIT = new Map()` should be created and the `checkFreeLimit`/`incrementFree` calls replaced with `checkReadmeLimit`/`incrementReadme`.

### 3.6 JavaScript Behavior

#### 3.6.1 Event Handling

| Event | Handler |
|-------|---------|
| Form submit / Generate button click | `generateReadme()` |
| Copy button click | `copyToClipboard()` |
| Download button click | `downloadReadme()` |
| Share button click | `shareReadme()` |
| Textarea input | Update character counter, enable/disable button |
| Language dropdown change | Store preference in `localStorage` |

#### 3.6.2 Core Functions

**`generateReadme()`**
1. Read form values (project name, language, code)
2. Validate: code must be non-empty
3. Show loading state (disable button, show spinner)
4. Call `POST /api/generate-readme` with JSON body
5. On success: render markdown preview, update rate limit badge, show action buttons
6. On error: display error message in result area
7. On rate limit (429): show warning with upgrade link
8. On completion: re-enable button

**`copyToClipboard()`**
1. Get the raw markdown text (from response data, not rendered HTML)
2. Use `navigator.clipboard.writeText()`
3. Show "рҹ“Ӣ Copied!" toast notification for 2 seconds
4. Fallback to `document.execCommand('copy')` if Clipboard API unavailable

**`downloadReadme()`**
1. Create a Blob from the raw markdown text
2. Create a temporary `<a>` element with `download="README.md"`
3. Set `href` to `URL.createObjectURL(blob)`
4. Trigger click, then revoke the object URL

**`shareReadme()`**
1. If `navigator.share` is available (mobile): `navigator.share({ title, text: readme, url })`
2. Otherwise: copy page URL to clipboard with toast feedback

#### 3.6.3 State Tracking

```js
const state = {
  currentReadme: '',     // Raw markdown from last generation
  remainingToday: 3,     // Tracked from API response headers
  isGenerating: false,   // Prevent double-submit
};
```

#### 3.6.4 Rate Limit Tracking

- On page load, if the user has already generated today, `remainingToday` starts at 3.
- Decremented after each successful generation from the `X-RateLimit-Remaining` header.
- Displayed in the hero badge.
- When 0, the Generate button shows "рҹ”’ Limit reached вҖ” try again tomorrow" and is disabled.
- Rate limit state is stored in `localStorage` (key: `readme-gen-remaining`, with date stamp) so it persists across page refreshes, and is validated against the server response.

## 6. Edge Cases

| Case | Handling |
|------|----------|
| Empty code field | Button disabled, error message shown |
| Code exactly 8000 chars | Allowed (boundary) |
| Code 8001+ chars | Truncation NOT allowed вҖ” return 413 error |
| Network failure during API call | Show "Network error. Please check your connection and try again." |
| DeepSeek API returns gibberish | Still returned as-is with attribution footer appended |
| Extremely long generated README | Preview area has `max-height: 500px` with scroll |
| User disables JavaScript | Page shows "JavaScript required" message via `<noscript>` tag |
| Web Share API not supported | Share button falls back to copying URL |
| Clipboard API not available | Copy button falls back to `execCommand('copy')` |
| Concurrent requests (rapid clicking) | `isGenerating` flag prevents double-submit |
