🚀 Activating my-automaton Gateway

One command brings the full AI agent API to life: x402 USDC payments, free tier, premium endpoints, referral system, and more.

📋 Current Status (as of container)

Serving: ✅ Static content (/, /playground.html, /api-docs.html, /comparison.html, etc.)
Missing: ❌ All API endpoints (/free/analyze, /v1/review, x402 USDC webhook, etc.)
Root cause: Running gateway.cjs is an OLD version. Production code at production-gateway.js is ready.

⚡ Step 1: Copy production code

sudo cp /root/automaton/production-gateway.js /root/automaton/gateway.cjs

This replaces the running gateway.cjs with the FULL version containing all API endpoints, x402 USDC integration, and badge system.

🚀 Step 2: Restart gateway service

sudo systemctl restart automaton-gateway

Or if your service is named differently:

sudo service automaton-gateway restart

Wait 2 seconds, then verify:

sleep 2 && curl -s http://localhost:8080/health | python3 -m json.tool

Expected response: {"status":"ok","version":"production-v1.0","x402":true,"deepseek":true}

🧪 Step 3: Verify everything works

Test all the key endpoints:

# Static content
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/ → 200
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/playground.html → 200

# Health
curl -s http://localhost:8080/health

# Free API (3/day per IP)
curl -s -X POST http://localhost:8080/free/review \
  -H 'Content-Type: application/json' \
  -d '{"code":"function bad(x){eval(x)}","language":"javascript"}'

# Premium API (needs API key)
curl -s -X POST http://localhost:8080/v1/security \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"code":"const db = sql`SELECT * FROM users WHERE id = ${userInput}`","language":"javascript"}'

# Stats
curl -s http://localhost:8080/api/stats/overview

# Badge
curl -s -o /dev/null -w "%{http_code}" "http://localhost:8080/badge?label=code%20review&message=PASS&color=green"

# Sitemap
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/sitemap.xml
