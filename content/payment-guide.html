🔧 How It Works

Step 1: Call the endpoint without payment

You'll get a 402 Payment Required response with payment details:

HTTP/1.1 402 Payment Required
Content-Type: application/json
X-Payment-Wallet: 0x76eADdEBFfb6A61DD071f97F4508467fc55dd113
X-Payment-Amount: 1
X-Payment-Currency: USDC
X-Payment-Chain: Base

{"error":"Payment required","price_cents":1,"wallet":"0x76eADdEBFfb6A61DD071f97F4508467fc55dd113"}
Step 2: Send USDC to the wallet on Base chain

Send exactly the required amount in USDC to 0x76eADdEBFfb6A61DD071f97F4508467fc55dd113 on Base network.

Tools you can use:

  • 🦊 MetaMask — add Base network, send USDC
  • ⚡ Coinbase Wallet — native Base support
  • 🔗 Base Bridge — bridge USDC to Base
  • Ledger, Trezor, or any EVM wallet
Step 3: Retry with payment header

Call the endpoint again with your transaction hash:

POST /v1/analyze
Content-Type: application/json
X-X402-Payment: 0xabc123...your_tx_hash

{"text":"Analyze this text for sentiment and key themes"}
Step 4: Get your results

If the payment is valid, you'll get your analysis:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "sentiment": "positive",
  "confidence": 0.92,
  "key_themes": ["innovation", "AI", "automation"],
  "summary": "..."  
}

❓ FAQ

What is x402?

x402 is a protocol for HTTP 402 payment. The server requests payment, you pay via USDC, and include the transaction hash on retry. Think of it as a pay-per-request API.

Which chain?

Base (Coinbase's L2). USDC only. Make sure you're on the Base network when sending.

Do I need an account?

No. Just send USDC from any wallet and retry with the transaction hash. No signup, no API keys.

Refunds?

Valid payments that we can verify will always return your results. If there's a system error, contact us and we'll process a refund in USDC.
