What is x402?

x402 is a protocol where APIs return HTTP 402 (Payment Required) with payment instructions instead of blocking access. You send the micro-payment, retry the request with the transaction hash, and get your result. It's like a vending machine for AI.

How It Works

Step 1: Make your API call normally. If payment is needed, you'll get HTTP 402 with details on how much to send and where.
Step 2: Send the required amount in USDC to the provided wallet address on Base chain (gas: < $0.001).
Step 3: Retry your request with the transaction hash as proof of payment. Get your result instantly.

JavaScript

async function callAnalyze(text) {
  const url = 'https://automation.songheng.vip/v1/analyze';
  
  // First attempt — will get 402 if payment needed
  let res = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ text, mode: 'analyze' })
  });
  
  if (res.status === 402) {
    // Send $0.01 USDC to wallet on Base chain
    const txHash = await sendUSDC(
      '0x76eADdEBFfb6A61DD071f97F4508467fc55dd113', 
      0.01 // 1 cent
    );
    
    // Retry with payment proof
    res = await fetch(url, {
      method: 'POST',
      headers: { 
        'Content-Type': 'application/json',
        'X-X402-Payment': txHash 
      },
      body: JSON.stringify({ text, mode: 'analyze' })
    });
  }
  
  return res.json();
}

FAQ

What chain?

Base (Ethereum L2). Transactions cost ~$0.0003 in gas.

What wallet?

Any wallet that supports Base and USDC: MetaMask, Coinbase Wallet, Rainbow, etc.

How do I get USDC?

Buy on Coinbase, Uniswap (Base), or bridge from Ethereum mainnet. Minimum purchase: ~$5.

What if I overpay?

Overpayment credits are tracked per wallet. Your next request will use the credit balance first.
