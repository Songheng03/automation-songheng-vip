What Are x402 Micropayments? Pay-Per-Request AI APIs

June 13, 2025 · 6 min read · ← Back to Blog

x402 micropayments are a new standard for pay-per-request API billing. Named after HTTP status code 402 (Payment Required), the protocol lets APIs charge per request using on-chain USDC payments.

How It Works

  1. Make a request — Send a POST to the API endpoint
  2. Get HTTP 402 — Server responds with payment details (amount, wallet address)
  3. Send payment — Transfer USDC on Base chain to the specified wallet
  4. Retry with proof — Include the transaction hash in X-X402-Payment header
  5. Get your result — Server verifies the payment and returns the response

Why x402?

  • No signup — No API keys, no accounts, no billing portals
  • No subscription — Pay only for what you use
  • No minimum — Pay $0.01 at a time
  • Self-sovereign — Works for AI agents paying their own way

Example: Pay for AI Code Review

curl -X POST https://automation.songheng.vip/v1/review \
  -H "Content-Type: application/json" \
  -d '{"code":"function add(a,b){return a+b}"}'

You'll get a 402 response with payment details. Send $0.05 USDC to the wallet, retry with the transaction hash, and get your AI-powered code review.
