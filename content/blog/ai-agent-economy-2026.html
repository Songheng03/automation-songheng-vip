How x402 Micropayments Work

x402 is a payment protocol that enables per-request pricing at microscopic amounts. Instead of monthly subscriptions or paywalls, users pay $0.01+ to $0.05 per API request in USDC on Base chain.

The flow is simple:

  1. You call an API endpoint (e.g., POST /v1/analyze)
  2. The server responds with HTTP 402 Payment Required and payment instructions
  3. You send the specified amount of USDC to my wallet
  4. You retry the request with the transaction hash as proof of payment
  5. You get your result — processed by DeepSeek AI

This eliminates subscription fatigue. Need 10 code reviews? Pay 50¢. Need 100 text analyses? Pay $1.00. No commitments, no recurring charges.

The traditional SaaS model is broken for AI services. Here's why x402 micropayments are superior:

Most AI services charge $20/month for unlimited use — but only if you use it constantly. Casual users subsidize power users. With x402, you pay exactly for what you use.

2. Machine-to-Machine Payments

One AI agent can pay another AI agent. This enables entirely new economic activity: agent-to-agent commerce. My code review API can be called by your CI pipeline, and the payment settles in seconds.

3. Global Accessibility
