What is x402?

x402 is a standard for HTTP 402 Payment Required responses in AI APIs. When a client calls your endpoint:

  1. If they have credits, you respond normally
  2. If they don't, you return HTTP 402 with payment instructions (wallet address, amount, chain)
  3. The client sends USDC and retries with a transaction hash header

It's the simplest payment model for agent-to-agent commerce.

Step 1: Set Up Your MCP Server

Create an MCP-compatible server that exposes your AI tools. Here's a minimal example using Node.js:

import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';

const server = new Server({
  name: 'code-review-mcp',
  version: '1.0.0',
}, {
  capabilities: { tools: {} }
});

server.setRequestHandler('tools/list', async () => ({
  tools: [{
    name: 'review_code',
    description: 'Review code for bugs, security issues, and best practices',
    inputSchema: {
      type: 'object',
      properties: {
        code: { type: 'string', description: 'Source code to review' },
        language: { type: 'string', description: 'Programming language' }
      }
    }
  }]
}));

server.setRequestHandler('tools/call', async (request) => {
  if (request.params.name === 'review_code') {
    const { code, language } = request.params.arguments;
    
    // Call your AI backend (e.g., via HTTP to a gateway with x402)
    const result = await analyzeWithPayment(code, language);
    
    return { content: [{ type: 'text', text: result }] };
  }
});

const transport = new StdioServerTransport();
await server.connect(transport);

Step 2: Add x402 Payment Support

Wrap your handler with payment checking logic. On the server side, you need:

function checkPayment(req) {
  // Check X-X402-Payment header
  const paymentTx = req.headers['x-x402-payment'];
  if (!paymentTx) return { status: 402, body: {
    payer: '0x76eADdEBFfb6a61DD071f97F4508467fc55dd113',
    chain: 'base',
    token: 'USDC',
    amount: '0.05',
    description: 'Send 0.05 USDC to the address above and retry with X-X402-Payment header'
  }};
  
  // Verify the transaction on-chain
  const verified = verifyTransaction(paymentTx, '0.05');
  if (!verified) return { status: 402, body: { error: 'Payment not confirmed' }};
  
  return { status: 200 }; // Proceed
}

Step 3: Deploy Behind a Gateway

For production, run your MCP server behind a gateway that handles:

  • Routing — Multiple services on one domain
  • Payment verification — Check credits/USDC before proxying
  • Rate limiting — Prevent abuse
  • Free tier — 3 free requests/day per IP to convert users
