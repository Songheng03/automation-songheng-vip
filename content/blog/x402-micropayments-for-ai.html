The subscription model is broken for AI. $20/month for a service you might use once. API keys with minimum spends. Tiered plans that don't match actual usage.

What is x402?

x402 is a payment pattern that uses HTTP status code 402 (Payment Required) and blockchain transactions to create pay-per-use APIs. Here's how it works:

  1. You send a request to a premium API endpoint
  2. If you haven't paid, the server responds with HTTP 402 and payment instructions
  3. You send the required amount in USDC to the server's wallet address
  4. You retry the request with the transaction hash as proof of payment
  5. The server verifies the payment and returns the result

The name "x402" comes from the pattern: execute request → get 402 → pay → retry.

Why USDC on Base?

USDC is a stablecoin — 1 USDC = ~$1 USD. No volatility. No price discovery required. Users know exactly what they're paying.

Base chain offers near-zero transaction fees (~$0.001 per transfer). For a 1-cent payment, the fee is 10% — manageable. On Ethereum mainnet, the same transaction might cost $5, making micropayments impossible.

The 402 Status Code

HTTP 402 has existed since HTTP/1.1 (RFC 2616, 1999) but was marked "reserved for future use." No one actually implemented it — until now.

For autonomous agents, 402 is the perfect signal. It's machine-readable, standard, and unambiguous:

HTTP/1.1 402 Payment Required
Content-Type: application/json

{
  "error": "payment_required",
  "message": "Send 1 USDC ($0.01) to 0x76eADdEBFfb6... on base",
  "wallet": "0x76eADdEBFfb6A61DD071f97F4508467fc55dd113",
  "chain": "base",
  "costCents": 1,
  "service": "analyze"
}

A client can parse this response, construct and send a payment transaction, and retry with the confirmation. No API keys. No OAuth. No signup flow.

Why This Matters for AI Agents

Autonomous agents need to consume services programmatically. Traditional payment flows (credit cards, invoices, subscription portals) are designed for humans. x402 is designed for machines.

  • No accounts — agents don't have email addresses or billing portals
  • No negotiations — price is fixed per request, published in the 402 response
  • No recurring charges — agents pay as they go, preserving their budget
  • Universal access — any agent with a wallet can use any x402 service
