Step 2: Create the GitHub Actions Workflow

In your repository, create a new file at .github/workflows/ai-code-review.yml:

name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  ai-review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Get changed files
        id: changed-files
        uses: tj-actions/changed-files@v44
        with:
          files: |
            **/*.js
            **/*.ts
            **/*.jsx
            **/*.tsx
            **/*.py
            **/*.go
            **/*.rs
            **/*.java

      - name: Run AI Code Review
        if: steps.changed-files.outputs.any_changed == 'true'
        env:
          API_KEY: ${{ secrets.MY_AUTOMATON_API_KEY }}
        run: |
          for file in ${{ steps.changed-files.outputs.all_changed_files }}; do
            echo "Reviewing: $file"
            
            # Read file content
            content=$(cat "$file" | jq -Rs .)
            
            # Send to my-automaton for review
            result=$(curl -s -X POST https://automation.songheng.vip/v1/review \
              -H "Content-Type: application/json" \
              -H "X-API-Key: $API_KEY" \
              -d "{\"code\":$content,\"language\":\"$(echo $file | rev | cut -d. -f1 | rev)\"}")
            
            # Post review comment
            echo "$result" | jq -r '.review' >> review-comments.md
          done

      - name: Post review summary
        if: steps.changed-files.outputs.any_changed == 'true'
        uses: marocchino/sticky-pull-request-comment@v2
        with:
          header: ai-code-review
          path: review-comments.md
⚠️ Important: Add your API key as a GitHub Secret. Go to Settings → Secrets and variables → Actions and add MY_AUTOMATON_API_KEY with your API key as the value. Never hardcode API keys in workflow files!

Advanced: Review Specific Languages Only

Want to review only specific languages? Modify the files filter in the workflow:

# Only review TypeScript and Python files
files: |
  **/*.ts
  **/*.tsx
  **/*.py

# Review everything except test files
files: |
  **/*
  !**/*.test.*
  !**/*.spec.*
  !**/tests/**

Advanced: Security Scanning on Every Push

You can also run security scans on every push (not just PRs). Create .github/workflows/security-scan.yml:

name: Security Scan

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run security scan
        env:
          API_KEY: ${{ secrets.MY_AUTOMATON_API_KEY }}
        run: |
          # Scan all source files
          find . -type f \( -name "*.js" -o -name "*.ts" -o -name "*.py" \) \
            ! -path "*/node_modules/*" ! -path "*/.git/*" | while read file; do
            
            content=$(cat "$file" | jq -Rs .)
            curl -s -X POST https://automation.songheng.vip/v1/security \
              -H "Content-Type: application/json" \
              -H "X-API-Key: $API_KEY" \
              -d "{\"code\":$content}" | jq -r '.vulnerabilities[]? | "⚠️ \(.severity): \(.description) in \(.file // "unknown")"' 
          done

Best Practices

  1. AI reviews are additive, not replacements. Keep human reviewers. AI catches the obvious stuff so humans can focus on architecture.
  2. Review the right files. Don't scan minified bundles, lock files, or generated code. Filter with the files pattern.
