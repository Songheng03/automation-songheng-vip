How x402 Micropayments Work

June 14, 2026 · 5 min read

x402 is a protocol that repurposes the rarely-used HTTP 402 "Payment Required" status code for modern cryptocurrency micropayments. Instead of API keys, subscriptions, or user accounts, you simply pay per request.

The Flow

Step 1: Make a request to any x402 endpoint. If you have free requests remaining, you get the response directly. If not, the server returns HTTP 402 with payment details.

HTTP/1.1 402 Payment Required
Content-Type: application/json

{
  "error": "Payment required",
  "cost": "$0.01 USDC",
  "wallet": "0x76eADdEBFfb6A61DD071f97F4508467fc55dd113",
  "chain": "base",
  "chainId": 8453
}

Step 2: Send the required amount in USDC to the specified wallet on Base chain. Using a wallet like Phantom, Coinbase Wallet, or Rabby, send 0.01 USDC to the agent's address.

Step 3: Retry the request with your transaction hash in the X-X402-Payment header. The server verifies the payment and returns your response.

curl -X POST https://automation.songheng.vip/v1/analyze \
  -H 'Content-Type: application/json' \
  -H 'X-X402-Payment: 0xabc123...' \
  -d '{"text":"Analyze this"}'

Why x402?

Traditional API billing requires accounts, API keys, rate limiting, and billing infrastructure. x402 eliminates all of that. Anyone with a wallet can use any x402-enabled service instantly. No signup, no personal data, no recurring subscriptions.

Why Base Chain?

Base is an Ethereum L2 with sub-cent transaction fees and wide wallet support. USDC is the native stablecoin, meaning the value is always 1:1 with USD. A $0.01 payment costs less than 0.0$0.01+ in gas fees.
