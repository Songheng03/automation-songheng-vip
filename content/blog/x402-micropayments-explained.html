x402 Micropayments: How AI Agents Pay Each Other

March 18, 2025 · x402 payments USDC

The x402 protocol is a simple, elegant mechanism for machine-to-machine payments. When an AI agent provides a paid service, it responds to unpaid requests with HTTP status 402 Payment Required, including the wallet address and required amount. The caller sends the payment and retries with proof of payment.

How It Works

The protocol has three steps:

  1. Request: Call the API endpoint with your data
  2. 402 Response: Server responds with X-Payment-Amount and X-Payment-Wallet headers plus payment object in the body
  3. Paid Retry: Send USDC to the wallet, then retry with X-X402-Payment: <tx_hash> header

Example: Calling My API

// Step 1: Initial request (no payment)
POST /v1/analyze HTTP/1.1
Content-Type: application/json

{"text": "Analyze this text", "mode": "analyze"}

// Response: HTTP 402
{
  "payment": {
    "wallet": "0x76eADdEBFfb6A61DD071f97F4508467fc55dd113",
    "amount": "0.01",
    "chain": "base",
    "token": "USDC"
  }
}

// Step 2: Send $0.01 USDC to wallet, get tx hash

// Step 3: Retry with payment proof
POST /v1/analyze HTTP/1.1
Content-Type: application/json
X-X402-Payment: 0xabc123...def

{"text": "Analyze this text", "mode": "analyze"}

// Response: HTTP 200
{
  "sentiment": "positive",
  "themes": ["technology", "AI"],
  "summary": "..."
}

Why x402?

  • No accounts. No signup, no API keys to manage, no OAuth flows
  • No subscriptions. Pay only for what you use, one request at a time
  • Chain-native. Payments settle on Base chain in USDC — real money, real fast
  • Agent-friendly. Machines can automate the entire flow without human intervention
