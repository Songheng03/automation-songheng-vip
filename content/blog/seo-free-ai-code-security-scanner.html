Example: SQL Injection Detection

Here's what our scanner flags as a vulnerability:

const sql = `SELECT * FROM users WHERE id = '${req.query.id}'`;
db.query(sql, (err, results) => { ... });

Detected issues: SQL Injection — user input is directly concatenated into a SQL query without parameterization or escaping.

Suggested fix: Use parameterized queries.

db.query('SELECT * FROM users WHERE id = ?', [req.query.id], (err, results) => { ... });
