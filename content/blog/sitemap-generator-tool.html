What Does a Sitemap Do?

A sitemap.xml file provides:

  • Complete URL listing — Every page you want indexed
  • Priority hints — Tell Google which pages matter most (1.0 = homepage, 0.3 = archives)
  • Update frequency — How often content changes (daily for blogs, monthly for about pages)
  • Last modified dates — When each page was last updated

Best Practices for XML Sitemaps

1. Priority matters. Only give 1.0 to your homepage. Blog posts should be 0.8, static pages 0.5, and archive pages 0.3.

2. Stay under 50,000 URLs. If you have more, use a sitemap index file that links to multiple sitemaps.

3. Keep it under 50MB. Each sitemap file must be less than 50MB uncompressed.

4. Submit to Google. After uploading to your site root, submit via Google Search Console.

5. Use IndexNow for Bing. For Bing and Yandex, use IndexNow for instant indexing.

Example: A Complete Sitemap

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>
    <lastmod>2025-06-14</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://example.com/blog/post-1</loc>
    <lastmod>2025-06-13</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
</urlset>
