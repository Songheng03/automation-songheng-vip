Why Automated Code Review?

Manual code review has scaling problems:

  • Time: A thorough review takes 30-60 minutes per PR
  • Inconsistency: Different reviewers catch different issues
  • Fatigue: Review quality drops after the 3rd PR in a row
  • Bottlenecks: Senior devs become gatekeepers

AI code review solves these problems by providing consistent, instant feedback on every change — without burning out your team.

Prerequisites

  • A GitHub repository (public or private)
  • Admin access to configure webhooks
  • A web browser

Step 1: Register Your Repository

Send a POST request to register your repo with the AI review service:

curl -X POST https://automation.songheng.vip/api/github/register \
  -H "Content-Type: application/json" \
  -d '{
    "repo": "your-username/your-repo",
    "secret": "choose-a-random-secret-string",
    "mode": "free"
  }'

Parameters:

  • repo — Your GitHub repository name (format: "owner/repo")
  • secret — A random string to secure your webhook
  • mode — "free" (3 reviews/day) or "premium" (unlimited, $0.01+/review)

You'll receive a response with your webhook URL:

{
  "ok": true,
  "webhook_url": "https://automation.songheng.vip/api/github/webhook",
  "repo": "your-username/your-repo",
  "mode": "free"
}

Step 2: Configure the GitHub Webhook

  1. Go to your repository on GitHub
  2. Click Settings → Webhooks → Add webhook
  3. In Payload URL, enter the webhook URL from step 1
  4. In Content type, select application/json
  5. In Secret, enter the same secret you used in registration
  6. Under Which events?, select Just the push event
  7. Click Add webhook

That's it! Every time you push code, the AI will automatically review the changed files.

Step 3: Trigger Your First Review

Push a commit to any branch:

git add .
git commit -m "feat: add user authentication"
git push

The AI reviews every new and modified file. Issues detected include:

  • Syntax errors and runtime bugs
  • Security vulnerabilities (SQL injection, XSS, hardcoded secrets)
  • Performance bottlenecks
  • Code style violations
  • Missing error handling
  • Memory leaks

FAQ

Is it really free?

Yes. You get 3 free reviews per day. No credit card needed.

Can I use it for private repos?

Yes. GitHub webhooks work with both public and private repositories.

What happens at 3 free reviews?

The webhook still fires but reviews are skipped until the next day. You can upgrade to premium anytime for unlimited reviews starting at $0.01 each.

Does it work with GitHub Actions?

You can use API keys in GitHub Actions for CI/CD integration. The webhook is simpler for most teams.

How is this different from CodeRabbit?

CodeRabbit focuses on PR-level reviews. This service reviews every push and costs significantly less ($0.01+ vs 25¢+ per review). Free tier has no credit card requirement.
