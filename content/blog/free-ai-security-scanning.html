The Most Common Vulnerabilities (That AI Catches)

🔴 SQL Injection — The #1 web vulnerability. Still present in ~8% of web apps despite being documented for 25+ years. Our scanner catches string concatenation in queries, unsanitized user input, and missing parameterized queries.
🔴 Cross-Site Scripting (XSS) — Found when user input is rendered without escaping. The AI checks for innerHTML assignments, direct interpolation in templates, and missing Content-Security-Policy headers.
🟡 Cross-Site Request Forgery (CSRF) — Missing anti-CSRF tokens on state-changing endpoints. Common in REST APIs that accept cookie-based auth without CSRF protection.
🟡 Authentication Flaws — Weak password policies, missing rate limiting on login, JWT without signature verification, hardcoded API keys, and session fixation vulnerabilities.
🟡 Data Exposure — Stack traces returned to users, excessive error details, sensitive data in URLs, and unencrypted PII in transit or storage.

Example: Vulnerable Express.js Endpoint

Paste this into the scanner:

app.get('/api/search', async (req, res) => {
  const { q } = req.query;
  const results = await db.query(
    `SELECT * FROM products WHERE name LIKE '%${q}%'`
  );
  res.json(results.rows);
});

The AI will flag:

  • 🔴 CRITICAL — SQL Injection: User input (q) is interpolated directly into SQL string
  • 🔴 CRITICAL — Missing Input Validation: No length/sanitization check on query parameter
  • 🟡 HIGH — No Rate Limiting: Search endpoint accessible without throttling — enables scraping
  • 🟡 MEDIUM — No Access Control: Anyone can search without authentication

And provide the fixed version using parameterized queries.

Tips for Effective AI Security Scanning

  • Scan early, scan often. Run the scanner on every PR, not just before deployment
  • Scan your dependencies too. Paste package.json, requirements.txt, or go.mod to check for known vulnerable packages
  • Scan configuration files. Dockerfiles, CI configs, and environment files often contain hardcoded secrets
  • Don't ignore LOW findings. They indicate technical debt that compounds over time
