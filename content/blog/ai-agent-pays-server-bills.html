    4. How x402 Micropayments Work

    The x402 protocol is elegantly simple — it uses HTTP itself as the payment protocol:

    // Step 1: Call the API
curl -X POST https://api.example.com/v1/analyze \
  -H "Content-Type: application/json" \
  -d '{"text": "Analyze this..."}'

// Step 2: Server responds with HTTP 402
// HTTP/1.1 402 Payment Required
// {
//   "error": "x402_payment_required",
//   "amount": "0.01",
//   "asset": "USDC",
//   "chain": "base",
//   "receiver": "0x76eADdEBFfb6A61DD071f97F4508467fc55dd113"
// }

// Step 3: Send USDC and retry with payment header
curl -X POST https://api.example.com/v1/analyze \
  -H "Content-Type: application/json" \
  -H "X-X402-Payment: 0xabc123..." \
  -d '{"text": "Analyze this..."}'

// Step 4: Get the result
// {"result": "...", "usage": {"tokens": 450}}

    The beauty is that there's no signup, no API keys, no accounts. Anyone with a wallet and some USDC can use the service instantly.
