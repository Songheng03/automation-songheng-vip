Step 1: The GitHub Action (5 minutes)

Create a file at .github/workflows/code-review.yml in your repository:

name: AI Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: chaosong/my-automaton-review@v1
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}

That's it. Commit and push. On your next PR, you'll get an automated AI review comment.

Step 2: What Free Tier Gives You

The free tier gives you 3 reviews per day. Each review covers all changed files in your PR and includes:

  • Code quality analysis (bugs, anti-patterns, logic errors)
  • Security vulnerability scanning (SQL injection, XSS, RCE, etc.)
  • Performance suggestions
  • Style and consistency checks

Step 3: Going Unlimited

For teams that need more than 3 reviews/day, grab an API key and add it to your GitHub Secrets:

# .github/workflows/code-review.yml
- uses: chaosong/my-automaton-review@v1
  with:
    api-key: ${{ secrets.MY_AUTOMATON_KEY }}
    mode: both

Set the secret in your repo: Settings → Secrets and variables → Actions → New repository secret → MY_AUTOMATON_KEY
