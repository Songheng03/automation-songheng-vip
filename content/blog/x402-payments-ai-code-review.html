x402 Payments: AI Code Review with USDC Micropayments

Published June 2024 · 5 min read

Traditional AI API pricing requires accounts, credit cards, and minimum deposits. x402 changes that — you pay per request with USDC on Base chain, no account needed.

What is x402?

x402 is a payment protocol where the server responds with HTTP 402 (Payment Required) and payment instructions. You send the exact amount in USDC, then retry with the transaction hash. It's like a vending machine for AI.

How It Works

# Step 1: Make a request
curl -X POST https://automation.songheng.vip/v1/review \
  -H "Content-Type: application/json" \
  -d '{"code":"function add(a,b){return a+b}"}'

# Step 2: Get 402 with payment details
# → {"error":"free_limit_reached","payment":{
#     "address":"0x76eADdEBFfb6A61DD071f97F4508467fc55dd113",
#     "chain":"Base","token":"USDC","amountCents":5}}

# Step 3: Send USDC, retry with tx hash
curl -X POST https://automation.songheng.vip/v1/review \
  -H "Content-Type: application/json" \
  -H "X-X402-Payment: 0xTX_HASH" \
  -d '{"code":"function add(a,b){return a+b}"}'

Why Base Chain?

Base is an Ethereum L2 with sub-cent transaction fees. Sending $0.05 USDC costs a fraction of a cent. It's fast, cheap, and widely supported by wallets like Coinbase Wallet, MetaMask, and Rainbow.
