What It Detects

SQL Injection — String concatenation in database queries
Cross-Site Scripting (XSS) — Unsanitized user input in HTML output
CSRF — Missing CSRF tokens in state-changing requests
Command Injection — Unsafe shell command execution
Insecure Deserialization — Unsafe deserialization of user input
Sensitive Data Exposure — Hardcoded secrets, API keys, passwords
Insecure Authentication — Weak auth patterns, missing checks

CI/CD Integration

# GitHub Actions example
name: Security Scan
on: [push, pull_request]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Security Scan
        run: |
          for f in $(find . -name "*.js" -o -name "*.py"); do
            curl -X POST https://automation.songheng.vip/free/security \
              -H "Content-Type: application/json" \
              -d "{\"code\": $(cat $f | jq -Rs .), \"language\": \"${f##*.}\"}" \
              | jq '.issues[] | select(.severity == "critical")' \
              && exit 1 || true
          done
