How x402 Payments Enable the Agent Economy

The HTTP 402 status code — "Payment Required" — was defined in the original HTTP specification in 1998. It was ahead of its time. The infrastructure for digital micropayments simply didn't exist yet. Twenty-seven years later, it finally has a use case: machine-to-machine payments in the agent economy.

The Problem: Paying for AI Services

Before x402, if an AI agent wanted to pay for another agent's service, the options were terrible:

  • API keys — require human registration, management, and billing accounts
  • Subscriptions — you pay whether you use the service or not
  • Credit systems — proprietary tokens that don't transfer between services
  • Smart contracts — too slow and expensive for penny-level transactions

None of these work for autonomous agents. Agents can't fill out signup forms. They can't manage billing dashboards. They need a protocol that works purely over HTTP.

How x402 Works

The x402 protocol solves this with a simple request-response-payment cycle:

  1. Request: Agent A sends a POST to Agent B's API endpoint
  2. 402 Response: Agent B returns HTTP 402 with payment details (amount, wallet address, chain)
  3. Payment: Agent A sends the exact amount in USDC via Base chain
  4. Retry: Agent A retries the request with the transaction hash in the X-X402-Payment header
  5. Fulfillment: Agent B verifies the payment and returns the result
POST /v1/analyze HTTP/1.1
Content-Type: application/json

{"text": "Analyze this..."}

Response:

HTTP/1.1 402 Payment Required
Content-Type: application/json

{
  "error": "Payment required",
  "costCents": 1,
  "payTo": "0x76eADdEBFfb6A61DD071f97F4508467fc55dd113",
  "chain": "base",
  "currency": "USDC"
}

Why Base Chain?

Base (Coinbase's Ethereum L2) is ideal for x402 payments because:

  • Low fees — fractions of a cent per transaction
  • Fast finality — confirmations in 1-2 seconds
  • USDC native — the standard stablecoin for machine payments
  • EVM compatible — works with standard Ethereum tooling
  • Growing ecosystem — millions of users and growing agent infrastructure

The Economics of 1-Cent Transactions

At $0.01 per transaction, x402 enables entirely new business models:

  • Analyze text: $0.01 per request
  • Summarize: $0.02 per request
  • Code review: $0.05 per request
  • Security scan: $0.03 per request
  • Content generation: 10¢ per article

These microprices add up. At 100 requests/day across 7 services, that's $5-15/day in potential revenue — enough to sustain a small agent on a $20/month VPS.

The Agent Economy is Here

x402 turns HTTP into a payment protocol. Any agent with a wallet can instantly pay any other agent for services. No signup. No approvals. Just HTTP requests and USDC settlements on Base chain.
