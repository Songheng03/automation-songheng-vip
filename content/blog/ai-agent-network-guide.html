Architecture Overview

Here's the stack I use:

Everything runs on a single VPS with no open ports except 8080. All service routing goes through a reverse proxy gateway.

Step 1: The Gateway Pattern

The key insight: one port, many services. Instead of exposing multiple ports, all traffic flows through a single HTTP server on port 8080 that routes to internal services.

// Gateway routing example
app.post('/v1/analyze', x402.middleware(0.01), services.analyze);
app.post('/v1/summarize', x402.middleware(0.02), services.summarize);
app.post('/v1/review', x402.middleware(0.05), services.review);
// Free routes are direct
app.post('/api/summarize', services.freeSummarize);

Step 2: USDC Micropayments (x402 Protocol)

I use the x402 payment protocol — it's like HTTP 402 "Payment Required" but with stablecoins:

  1. Client calls an endpoint normally
  2. Server responds with HTTP 402 + payment details (amount, wallet address, chain)
  3. Client sends USDC to the server's wallet on Base chain
  4. Client retries with X-X402-Payment: tx_hash header
  5. Server verifies the transaction and serves the response
Why Base chain? Fast (2s finality), cheap ($0.001/tx), USDC native. No gas token required for USDC transfers.
