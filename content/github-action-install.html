Create the workflow file

In your repository, create .github/workflows/ai-code-review.yml

name: AI Code Review
on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write
  issues: write

jobs:
  review:
    runs-on: ubuntu-latest
    name: AI Code Review
    steps:
      - uses: actions/checkout@v4
      
      - name: AI Code Review
        uses: my-automaton/mcp-server@main
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          mode: review
          
      - name: Code Quality Badge
        uses: my-automaton/mcp-server@main
        with:
          mode: badge
          output: .github/badges/quality.svg

Commit and push

Commit the file to your default branch. That's it — every new PR will automatically get an AI review comment.

git add .github/workflows/ai-code-review.yml
git commit -m "Add AI code review"
git push

❓ FAQ

What does the AI review check?

Bug detection, security vulnerabilities (OWASP Top 10), performance issues, code style violations, anti-patterns, and best practices. It covers JavaScript, TypeScript, Python, Go, Rust, Java, and 20+ other languages.

Is it really free?

Yes! 3 free reviews per day per IP address. No credit card, no signup. For unlimited reviews or faster processing, upgrade from $5/month.

What permissions does it need?

The action needs pull-requests: write to post review comments and contents: read to access your code. Your code is never stored — it's sent to the AI API for analysis and discarded after review.

Can I use it with a monorepo?

Yes! The action analyzes each PR's diff, which works with any repository structure including monorepos. It reviews only changed files.
