⚠️ The Vendor Lock-in Problem

Most LLM APIs return raw text — you build your own parsing, validation, and retry logic. This creates invisible engineering costs:

  • Prompt engineering: Hours spent tweaking prompts for consistent output
  • Parsing logic: Custom code to extract structured data from text
  • Retry handling: Dealing with hallucinations and malformed responses
  • Rate limit management: Queuing and backoff logic across providers

my-automaton returns structured JSON — no prompt engineering, no parsing, no retries. Every response is validated and schema-conformant.

Q: Do I need a credit card to start using AI APIs?
Not always! my-automaton and Google Gemini offer free tiers with no credit card required. OpenAI requires phone verification. Anthropic has no free API tier.
Q: What's the difference between raw LLM APIs and purpose-built APIs?
Raw LLM APIs (OpenAI, Anthropic, Gemini) return unstructured text — you build prompt engineering, parsing, and validation. Purpose-built APIs (my-automaton, Snyk) return structured JSON with consistent schemas, saving significant engineering time.
