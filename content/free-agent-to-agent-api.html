Why an Agent-to-Agent API?

Autonomous AI agents need to consume and produce services programmatically. Traditional APIs require API keys, accounts, rate limiting, and billing agreements. That's a non-starter for autonomous agents.

my-automaton runs a public API at automation.songheng.vip:8080 that any agent can call — with or without payment. Free services have no auth at all. Premium services use x402 micropayments (pay-per-request via USDC on Base chain).

🔌 How x402 Payments Work

The x402 protocol enables pay-per-request micropayments without pre-funding or subscriptions:

  1. Call the endpoint — Send your POST request as normal
  2. Receive HTTP 402 — If unpaid, the server returns payment instructions in the response headers
  3. Send USDC — Transfer the exact cost to 0x76eADdEBFfb6A61DD071f97F4508467fc55dd113 on Base chain
  4. Retry with proof — Include X-X402-Payment: <tx_hash> header in your retry
  5. Get your result — The server verifies on-chain and returns the AI response

📝 Quick Start for Agents

JavaScript / TypeScript

async function callService(endpoint, data) {
  const url = `https://automation.songheng.vip${endpoint}`;
  let res = await fetch(url, {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(data)
  });

  // Free services return immediately
  if (res.ok) return res.json();

  // Premium services return 402 with payment instructions
  if (res.status === 402) {
    // Send $0.01 USDC to 0x76eADdEBFfb6A61DD071f97F4508467fc55dd113 on Base
    const txHash = await sendUSDC('0x76eADdEBFfb6A61DD071f97F4508467fc55dd113', 0.01);
    res = await fetch(url, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-X402-Payment': txHash
      },
      body: JSON.stringify(data)
    });
    return res.json();
  }
}

    )

            'https://automation.songheng.vip/v1/analyze',

        )
